Лабораторна робота №2.2

Тема: Адаптивна верстка. Медіа-запити. Метатег viewport

Мета роботи: навчитися створювати адаптивну верстку за допомогою CSS медіа-запитів, зрозуміти принципи використання метатегу viewport, медіа-типів і функцій, а також застосувати стратегію Mobile First.

1. Десктопна версія

Десктопна версія відображає повний контент сторінки з горизонтальним меню, блоками у ряд та великими зображеннями. Для неї використовується стандартний стиль без медіа-запитів.

2. Планшетна версія

При ширині до 768–1024px елементи перебудовуються у два рядки. Меню стає вертикальним, а зображення змінюють масштаб для кращого відображення.

@media (max-width: 1024px) { .container { flex-direction: column; } .sidebar { display: none; } }

3. Мобільна версія

Для пристроїв із шириною до 480px використовується повністю вертикальна структура. Блоки займають 100% ширини екрана, збільшується розмір шрифту для зручності користування.

@media (max-width: 480px) { body { font-size: 16px; } nav a { display: block; padding: 10px; } }

4. Метатег viewport

Метатег viewport використовується для налаштування масштабу сторінки на мобільних пристроях:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Цей тег гарантує, що сторінка відображатиметься коректно на всіх екранах без горизонтальної прокрутки.

5. Логічні оператори в медіа-запитах

Медіа-запити підтримують логічні оператори and, or, not для комбінування умов. Наприклад:

@media screen and (min-width: 600px) and (max-width: 900px) { .container { background: #222; } }

Висновки

У ході виконання лабораторної роботи створено три версії веб-сторінки: десктопну, планшетну та мобільну. Застосовано медіа-запити, логічні оператори, а також метатег viewport. Це дозволило забезпечити коректне відображення сторінки на різних пристроях та екранах.
